** Graphics driver policy (stock cmdline today)

The live system forces Iris Xe on =i915= and probes =xe= only for PCI ID 9a49:

#+begin_example
i915
i915.force_probe=!9a49
xe.force_probe=9a49
#+end_example

linux-rg carries both =CONFIG_DRM_I915=m= and =CONFIG_DRM_XE=m= so either path
can be selected at boot without rebuilding. Document the chosen default in
=linux-rg-i915= after first linux-rg boot.

** What this hardware is not

| Class | Why skip |
| AMD znver / x86-64-v3 ISA patches | Intel Raptor Lake-U, not AMD |
| Panther Lake PSR / Panel Replay hunks | Wrong display IP generation |
| Handheld / Steam Deck patches | Wrong form factor |
| Large-server SMT throughput tuning | 2-thread SMT on one socket; laptop latency matters more |
| cgroup VRAM / rt-i915 | iGPU laptop; patches do not apply or PREEMPT_RT not enabled |
| MUQSS / full CONFIG_CACHY fair | Superseded by EEVDF+BORE+sched_ext stack |

* Workload profiles

Targets are scored per profile. Default daily driver: =undocked-mixed=.

| Profile | Typical load | Priority metrics |
| undocked-mixed | Sway, browser, terminals, sync, occasional =make -j9= | wakeup latency, scroll jank, fan noise, idle power |
| docked-office | External 4K via TB4, DDCCI brightness, Ethernet/USB dock | hotplug reliability, display wake, DDCCI, idle with dock |
| travel | Wi-Fi roaming, VPN, suspend in bag | TCP setup latency, s2idle resume, battery drain |
| build-heavy | =make -j9=, containers, scientific builds | build wall time, PSI memory, NVMe contention under desktop |
| memory-pressure | browser + VM + build concurrently | swap thrash, oomd reaction, zram compression ratio |

* Subsystem targets

** Hybrid CPU and scheduler (i7-1365U)

| Target | Rationale | Kernel / userspace | Vissue |
| Fair-class desktop latency under mixed load | 12-thread hybrid shares P/E clusters | BORE, detach_tasks, migration_cost 300us | =linux-rg-h2e3=, =linux-rg-78xs= |
| Load-balance correctness on P/E | Frequent migration across clusters | detach_tasks fix (0011) | =linux-rg-78xs= |
| sched_ext policy without fair rewrite | Literature grade B; reversible | CONFIG_SCHED_CLASS_EXT, scx_loader, ASA heuristic | =linux-rg-tjsl=, =linux-rg-asa1= |
| SMT sibling locality on wake | 2-way SMT per core; not a 64-thread server | sched_ext SMT idle (0012) | =linux-rg-9k2m= |
| Context-switch hot path | Kolivas-adjacent, upstream-reviewed inlines | 0010 hot-path bundle | =linux-rg-q3np= |
| cache-aware SCHED_CACHE | LLC placement experiment | Hold until scx_loader A/B fails | =linux-rg-kr7t= |

Acceptance (post-boot, =linux-rg-bench-sched=):

- =make -j9= wall time within 5% of stock Arch kernel or faster.
- No sustained scheduler stalls visible in =/proc/pressure/cpu= during desktop use.
- scx_loader trial documents chosen expert vs BORE-only for =undocked-mixed=.

** NVMe and block I/O (KIOXIA XG8)

| Target | Rationale | Kernel | Vissue |
| Interactive I/O under build + browser | Single NVMe serves page cache, builds, sync | BFQ + contention patch (0007) | =linux-rg-6myb= |
| Optional lower-latency mq scheduler | HOTSTORAGE 2023 questions default mq on ULL NVMe | ADIOS module, non-default | =linux-rg-rt1t= |
| ASPM on PCIe | Laptop on battery | CONFIG_PCIEASPM_POWER_SUPERSAVE | =linux-rg-wpdr= |

Acceptance (=linux-rg-rt1t=):

- Compare =mq-deadline= (bfq for single-queue legacy N/A), =bfq=, =adios=, =none=.
- Do not set ADIOS default without two consecutive wins on this SSD.

* Patch and policy priority (this machine)

Ordered by expected impact on daily =undocked-mixed= profile:

1. Bootable package + rEFInd rollback (=linux-rg-py5f=, =linux-rg-mxx8=)
2. Scheduler stack already carried (0001-0013, 0009 ASA)
3. NVMe / BFQ benchmark (=linux-rg-rt1t=)
4. scx_loader + ASA heuristic eval (=linux-rg-tjsl=)
5. zram + oomd policy for 32 GiB (=linux-rg-zram=, =linux-rg-oomd=)
6. Idle power / cpuidle (=linux-rg-7m20=, =linux-rg-pwr=)
7. Memory vmscan weld if pressure tests fail (=linux-rg-7zwz=, =linux-rg-5fxc=)
8. cache-aware branch only if sched_ext insufficient (=linux-rg-kr7t=)
