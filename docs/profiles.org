#+title: linux-rg machine profiles

* Purpose

This is the index of machine profiles supported (or in development) for the
linux-rg kernel package. It is the starting point for "which patches and areas
apply to which profile".

Shared assets (apply or are evaluated for all profiles unless noted):
- Patch series at repo root (0001-*.patch and adaptation patches)
- =docs/literature-review.org= (evidence grades, venue map)
- =docs/candidates.org= (candidates + applicability table per entry)

Per-profile assets:
- =${slug}.config= overlay applied during prepare
- =docs/${slug}-kernel.org= -- profile definition + why specific patches are carried
- =docs/${slug}-targets.org= -- verified hardware, workload profiles, measurable targets, vissues
- =docs/${slug}-drivers.org= -- driver map and firmware notes
- =docs/${slug}-boot-safety.org= -- boot contract, mkinitcpio, rollback rules
- =docs/${slug}-userspace.org= -- host policy, root helper workflow, postboot verification (when present)
- Profile-specific check scripts (e.g. =rgx1gen11-*-check=)

* Patch and area discovery across profiles

When evaluating a new patch or policy area:

1. Start in =docs/literature-review.org= for evidence grade.
2. Record motivation + measured risk in =docs/candidates.org=.
3. Add an applicability row (or update existing) with the profiles it touches.
4. For each profile that may carry it:
   - Update that profile's targets.org (why it matters here, acceptance criteria).
   - Update that profile's kernel.org (fit, tradeoffs, config impact).
   - If hardware-specific validation exists, extend or add a checker.

A patch can be:
- Carried for all active profiles (rare for low-level hardware tweaks).
- Carried for a subset (common).
- Carried as a module for one profile, off for another.
- Evaluated and blocked for a profile with a note.

* Naming conventions

- Profile slugs are short, lowercase, no separators when possible (rgx1gen11, rgam5terra, rgSURFLat).
- All per-profile files in docs/ use the =${slug}-= prefix.
- The machine overlay in the repo root is =${slug}.config=.
- Checker / helper scripts that ship in the package are currently =${slug}-*= (this
  will be generalized when the profile selector lands).

* Status and process

New profiles are added only after:
- Primary-source research against the actual hardware (or credible public
  documentation + reproducible behavior on the target).
- A targets.org with verified inventory and workload definitions.
- A kernel.org section explaining patch applicability for that silicon/family.
- Vissue(s) tracking the work.

Installer and Archiso selector work consumes the profile contract instead of
guessing from receipt data alone.
