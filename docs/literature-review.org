#+title: linux-rg literature review (rgx1gen11)

* Scope

This document records research-grade sources used to evaluate kernel choices for
the ThinkPad X1 Carbon Gen 11 profile. It complements the patch rationale in
=docs/rgx1gen11-kernel.org= and candidate notes in =docs/candidates.org=.

Hardware context:

- Intel Core i7-1365U, Raptor Lake-P/U hybrid (P-cores and E-cores).
- KIOXIA XG8 NVMe (ultra-low-latency class storage).
- Intel Iris Xe integrated graphics (display version below Lunar Lake).
- Intel CNVi Wi-Fi, Thunderbolt 4 docking.

Sources are limited to peer-reviewed venues, IETF drafts, kernel.org and LWN
primary coverage, USENIX/LPC proceedings, and vendor technical documentation.
Downstream patch aggregators are not treated as evidence on their own.

* Evidence grading

| Grade | Meaning | Examples |
| A | Peer-reviewed or standards-track | USENIX, ACM, IMC, PAM, HOTSTORAGE, IETF drafts |
| B | Primary kernel engineering literature | kernel.org docs, LWN, LPC slides with published materials |
| C | Vendor or production engineering reports | Intel, Lenovo, Meta/Facebook engineering notes |
| D | Downstream implementation without independent study | Single-author patch stacks without published evaluation |

- IETF draft: https://datatracker.ietf.org/doc/draft-ietf-ccwg-bbr/
- PAM 2024: Promises and Potential of BBRv3
  https://research.vu.nl/en/publications/promises-andpotential-ofbbrv3/
- MPI BBRv3 evaluation portal
  https://inet-bbrv3eval.mpi-inf.mpg.de/

Evidence grade: A/B.

Research implication: BBRv3 is literature-supported for variable-RTT paths
(Wi-Fi, VPN, roaming SSH). Fairness risk remains documented; keep Cubic as an
explicit fallback on shared-buffer networks.

* Block I/O and NVMe

** BFQ (carried in linux-rg)

- Valente and Avanzini, ACM OSR 2012
  https://dl.acm.org/doi/10.1145/2367589.2367590
- Valente et al.: Evolution of the BFQ Storage I/O Scheduler
  https://iris.unimore.it/bitstream/11380/1108855/2/ValenteEvolution.pdf
- Kernel documentation: https://docs.kernel.org/block/bfq-iosched.html

Evidence grade: A for BFQ on shared and rotational workloads.

** Ultra-low-latency NVMe (critical for KIOXIA XG8)

- Whitaker et al., HOTSTORAGE 2023: Do we still need IO schedulers for
  low-latency disks?
  https://www.hotstorage.org/2023/papers/hotstorage23-final1.pdf

Evidence grade: A.

Key finding: on ULL storage, mq schedulers often fail to improve performance
or energy efficiency and can increase latency. Schedulers may still matter for
fairness and QoS, but default scheduler use on ULL NVMe is an empirical
question, not an assumption.

** Cachy memory ratios, mglru-patches-v2, cache-aware (candidates)

- Cachy =vm.anon_min_ratio= and clean ratio knobs: grade B concept / D patch.
  Kconfig help cites zram thrash; aligns with SWAM/ATC framing. Extract without
  =CONFIG_CACHY=; preferred weld partner for mglru-patches-v2.
- mglru-patches-v2: grade D as downstream dev-tree port; complements ratio knobs
  on =mm/vmscan.c=; do not stack with re-swappiness without manual merge.
- cache-aware (=CONFIG_SCHED_CACHE=): grade D; hybrid LLC motivation per
  Brown-Neri LPC 2022 but scheduler-only and conflicts with BORE.

Welding map: =docs/candidates.org=.

* Synthesis for rgx1gen11

| Component | Literature support | Action |
| MGLRU, PSI, memcg, zswap, zram, DAMON | Strong (A/B) | Keep; tune via userspace policy |
| sched_ext enabled | Strong (B) | Keep; evaluate scx schedulers empirically |
| BBRv3 default | Strong with fairness caveats (A) | Keep; document Cubic fallback |
| BFQ plus block contention | BFQ historically strong (A); ULL NVMe benefit uncertain (HOTSTORAGE 2023) | Benchmark against =none= on KIOXIA XG8 |
| ADIOS module | No published evaluation (D) | Carry as module only; measure before defaulting |
| BORE | No published evaluation (D) | Highest-risk carried scheduler patch |
| re-swappiness, cpuidle TEO, detach_tasks | Problem framed in A-tier work; patches are D until measured | Document as candidates; benchmark before carry |
