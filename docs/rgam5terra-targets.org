** Thermal constraint (primary research finding)

The Terra case plus NH-L9a-AM4 physically mounted on AM5 is a **low-profile air cooler on a
170 W CPU** in a volume-constrained chassis, paired with a **~250 W-class
discrete GPU**. Linux power/thermal policy is a first-class target here, not an
afterthought. Expect userspace caps (EPP, PPT, fan curves, GPU power limits) to
matter as much as kernel scheduler choices.

** What this hardware is not (rgam5terra)

| Class | Why skip |
| Hybrid Intel sched weld (0010-0013) | No P/E clusters; Zen 5 is homogeneous per CCD |
| intel_pstate / intel_idle / TEO 0016 | Wrong CPU vendor |
| iwlwifi / CNVi / AX211 checkers | Wrong Wi-Fi stack |
| i915 / xe / Iris Xe boot contract | Discrete NVIDIA primary display |
| DDCCI in-tree backlight | No internal laptop panel; external monitor DDC is userspace |
| ThinkPad ACPI / s2idle ultrabook path | Desktop AM5; different ACPI sleep story |
| Panther Lake PSR patches | Wrong display IP |

* Thermal management spec (Terra + 9950X + NH-L9a + RTX 5070)

Full stack for managing heat in a ~12 L sandwich case. Shorter cables, fan
direction, and power caps matter as much as kernel tuning. Vissues: =linux-rg-1s03=
(T1), =linux-rg-t49m= (T8), =linux-rg-ncz9= (T35), =linux-rg-uxqx= (Slurm caps).

** Heat budget (why this dominates)

| Source | Stock envelope | Realistic Terra sustained target |
| CPU 9950X | 170 W TDP / 230 W PPT ceiling | *95-120 W PPT* (NH-L9a class) |
| GPU RTX 5070 Dual | ~250 W TGP class | *150-200 W* until measured (=T8=) |
| VRM + RAM + NVMe + PSU loss | 15-40 W under load | same order |
| Case | Fractal Terra sandwich; wood side panels | closed-panel =thermal-sustained= profile |

The sandwich isolates GPU and CPU chambers, but the *center spine and PSU zone
still share heat*. Uncapped CPU + GPU together will throttle something; design
for *one hot domain at a time* or accept lower clocks.

** Layer 1 - Physical build (do at assembly)

*** Cable and clutter (includes shorter cables)

| Item | Spec / action | Why |
| PSU cables | Use *short modular cables* for SP850 (custom length 15-25 cm segments) or route stock cables with *zero coil overlap* on GPU chamber intake | Blocked intake is the #1 Terra thermal failure mode |
| 12VHPWR / PCIe GPU power | Single clean path along case floor or spine; *no cable sitting on GPU fan intake* | ASUS Dual fans are bottom-biased |
| Front-panel / USB-C | Route along spine, not across CPU cooler intake | NH-L9a needs direct fin airflow |
| Wi-Fi/BT antenna wires | Tape to spine, away from exhaust | Avoid turbulence at P12 |
| PCIe 4.0 riser | Use *stock Fractal riser*; gentle bend radius; screw bracket firmly | Poor seating raises GPU temp + replay errors |
| NVMe | SN850X has heatsink label; ensure *thermal pad contacts*; do not stack cables on drive | Hot spot in CPU chamber |

Shorter cables are worth it: SP850 is modular (receipt). Budget *$30-80* for an
SFF cable kit sized for Terra + ITX, or measure and order per-length.

*** Fans and airflow

| Fan | Role | Draft spec |
| Arctic P12 Pro (receipt) | Case fan on Terra bottom bracket | *Intake* cool air into GPU or CPU chamber per Fractal manual; confirm orientation with smoke test |
| ASUS RTX 5070 Dual | 2x axial, stock curve | Stock until baseline temps; then *custom fan curve in nvidia-settings* or CoolerControl |
| Lian Li SP850 V2 | 120 mm internal PSU fan | PSU *fan down / intake from case* if manual allows; avoid recirculating hot GPU exhaust |
| NH-L9a-AM4 | 92 mm top-down on CPU | *No case fan fighting it*; ensure fins clear VRM; quality TIM (NT-H2 included) dot method |

Fractal Terra supports bottom 120 mm mount (=T23=). Common SFF patterns:

- *GPU-heavy gaming:* bottom fan as intake to GPU chamber; accept CPU relies on NH-L9a + chimney effect.
- *Build-heavy CPU:* bias intake toward CPU chamber side; cap PPT harder if GPU idle.

Run =thermal-sustained= with *wood panels on* (real use) and once with *panel
open* (diagnostic only) to separate airflow vs power-limit issues.

** Layer 5 - Measurement and acceptance

| Sensor | Command / tool | Alert threshold (draft) |
| CPU | =sensors=, =ryzenadj --info= | package > 95 C sustained |
| GPU | =nvidia-smi dmon=, =sensors= | hotspot > 83 C sustained |
| NVMe | =nvme smart-log=, =sensors nvme-pci*= | composite > 70 C |
| VRM | board sensor if exposed | > 90 C |
| Throttle | =journalctl -k -g THRM=, =nvidia-smi -q -d PERFORMANCE= | any sustained throttle in =thermal-sustained= |

=thermal-sustained= acceptance: 10 min combined load (e.g. =stress-ng --cpu= +
GPU furmark or game) with panels on; document clocks, noise, and whether PPT/GPU
PL caps were hit.

** Layer 6 - Hardware escape hatches (if spec insufficient)

Ordered by invasiveness:

1. *Shorter cables + fan curve tune* (free/cheap)
2. *Lower PPT / GPU PL* (free, performance cost)
3. *Second P12* on bottom bracket if Terra mount allows (receipt has one; buy second)
4. *Cooler swap*: NH-L12S / IS-55 class if height clears GPU (measure Terra GPU length mode)
5. *CPU downgrade in BIOS*: not ideal; prefer PPT cap on 9950X
6. *External GPU*: not part of this build profile

Kernel patches (BORE, znver5, gaming-sched) are *layer 7*; only after layers
1-5 are stable (=linux-rg-1s03= gate).
