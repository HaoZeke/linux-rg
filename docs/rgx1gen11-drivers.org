#+title: linux-rg rgx1gen11 driver map
#+date: [2026-06-18 Thu]

* Purpose

Kernel driver stacks for the rgx1gen11 profile (21HM002GUS), verified on the
bench 2026-06-18. Use with =docs/rgx1gen11-boot-safety.org= before first boot
on this hardware.

See =docs/profiles.org= for the full profile list.

Other profiles have (or will have) their own drivers.org. Shared driver-related
research lives in =docs/literature-review.org= and =docs/candidates.org=.

* Wi-Fi (Intel CNVi AX211 class)

| Field | Value |
| PCI | 8086:51f1 (Raptor Lake PCH CNVi WiFi) |
| Kernel modules | =iwlmvm= -> =iwlwifi= -> =mac80211= -> =cfg80211= |
| Firmware | =iwlwifi-ma-b0-gf-a0-*.ucode= + =.pnvm= from linux-firmware |
| Regulatory | wireless-regdb + crda/country code |
| RF kill | =thinkpad_acpi=, =rfkill=, =CONFIG_WIRELESS_HOTKEY=m= |

** Kernel config pins (rgx1gen11.config)

- =CONFIG_IWLWIFI=m=, =CONFIG_IWLMVM=m= (MVM for 802.11ac/ax/be)
- =CONFIG_IWLMLD=m= (multi-link device support on 7.0)
- =CONFIG_IWLWIFI_LEDS=y=
- Do not disable =CONFIG_CFG80211=m= or =CONFIG_MAC80211=m= (Arch base).

** Module parameters (shipped modprobe.d)

File: =/usr/lib/modprobe.d/rgx1gen11-iwlwifi.conf=

- =bt_coex_active=1: Wi-Fi/BT antenna coexistence on CNVi + USB BT combo.
- =power_save=: default module Y; use =iw dev wlan0 set power_save off= for
  low-latency desktop, =on= for battery (linux-rg-pwr).

** Firmware versions on bench (2026-06-18)

Packages present under =/lib/firmware/=:

- =iwlwifi-ma-b0-gf-a0-86.ucode.zst=
- =iwlwifi-ma-b0-gf-a0-89.ucode.zst=
- =iwlwifi-ma-b0-gf-a0.pnvm.zst=

Record the loaded microcode revision from =dmesg | rg iwlwifi= after each
=linux-firmware= bump and linux-rg boot (=linux-rg-wifi=).

** Userspace dependencies

- =linux-firmware= (required)
- =wireless-regdb= (recommended)
- iwd with iwctl

** Mechanical check

=rgx1gen11-ax211-check= validates the AX211 contract from either
=rgx1gen11.config=, a prepared kernel =.config=, an installed module tree, or
the running linux-rg boot:

#+begin_example
./rgx1gen11-ax211-check
./rgx1gen11-ax211-check --config /path/to/linux-7.0.12/.config
rgx1gen11-ax211-check --live
#+end_example

** Rust firmware loader

Arch 7.0.12 builds =CONFIG_RUST_FW_LOADER_ABSTRACTIONS=y= (pinned in
=rgx1gen11.config=). This is a safer firmware-loader API layer; iwlwifi remains
a C driver. See =docs/candidates.org= Rust subsection and =linux-rg-rust1=.

** What we are not doing

- No out-of-tree iwlwifi patches (boot risk, rebase cost).
- No firmware baked into =CONFIG_EXTRA_FIRMWARE=.
- No built-in (=y=) iwlwifi.

Vissues: =linux-rg-wifi=, =linux-rg-bench-wifi=.

* Bluetooth (Intel AX211 USB)

| Field | Value |
| USB | 8087:0033 Intel AX211 Bluetooth |
| Driver | =btusb= with =btintel= / =btmtk= helpers |
| ThinkPad switch | =thinkpad_acpi= rfkill for =tpacpi_bluetooth_sw= |

** Kernel config pins

- =CONFIG_BT=m=, =CONFIG_BT_INTEL=m=, =CONFIG_BT_HCIBTUSB=m=
- =CONFIG_BT_HCIBTUSB_AUTOSUSPEND=y=, =CONFIG_BT_HCIBTUSB_POLL_SYNC=y=
- =CONFIG_BT_HCIUART=m= + =CONFIG_BT_HCIUART_INTEL=y= (Arch default; not the
  active path on this box, but kept for ACPI UART fallback variants).
- =CONFIG_BT_INTEL_PCIE= disabled; the target adapter is USB =8087:0033=.

This machine uses *USB* BT, not =CONFIG_BT_INTEL_PCIE=. Do not disable =btusb=.

** Module parameters

File: =/usr/lib/modprobe.d/rgx1gen11-btusb.conf=

- Autosuspend enabled via Kconfig; document =hciconfig= / =bluetoothctl= after
  resume if devices disappear (linux-rg-slp).

** Firmware versions on bench (2026-06-18)

Stock =7.0.12-arch1-1= loaded:

- =intel/ibt-0040-0041.sfi= (firmware version 202-5.26)
- =intel/ibt-0040-0041.ddc=

The helper requires both files when run with =--require-firmware= or =--live=.

** Postboot acceptance

#+begin_example
bluetoothctl show
bluetoothctl devices Connected
journalctl -k -b --no-pager -g 'Bluetooth: hci0|btusb|8087'
#+end_example

The linux-rg boot passes the static BT gate when =btusb= and =btintel= are
loaded from the linux-rg module tree, the Intel firmware load has no =-ENOENT=,
and one paired device reconnects after one =systemctl suspend= cycle.

Vissue: =linux-rg-bt=.

* Fingerprint (Synaptics Prometheus)

| Field | Value |
| USB | 06cb:00fc Synaptics Prometheus MIS |
| Userspace | =fprintd= + =libfprint= (not active on bench 2026-06-18) |
| Kernel | USB HID stack; no custom kernel driver needed |

Prometheus readers on recent ThinkPads often need a current =libfprint= build and
matching udev rules. This is userspace-only for linux-rg; validate enroll/auth
after stock boot stability.

Vissue: =linux-rg-1j07=.

* Webcam (Chicony UVC)

| Field | Value |
| USB | 04f2:b751 Chicony integrated camera |
| Driver | =uvcvideo= (loaded) |
| Privacy | ThinkPad camera shutter / kill switch via ACPI where exposed |

Kernel pins: none beyond USB stack and =CONFIG_MEDIA_CAMERA_SUPPORT= (Arch base).
Policy focus: autosuspend, shutter integration, PipeWire portal permissions.

Chezmoi policy installs =/etc/udev/rules.d/70-rg-uvc-autosuspend.rules= for
=04f2:b751= so the integrated camera uses USB runtime PM =auto= with a 2000 ms
autosuspend delay. Validate the live policy with:

#+begin_src shell
rg-uvc-camera-check --check
#+end_src

Vissue: =linux-rg-al83=.

* Storage (KIOXIA XG8)

| Field | Value |
| PCI | 1e0f:0010 KIOXIA XG8 |
| Driver | =nvme=, =nvme_core= |
| Scheduler | BFQ default; ADIOS optional module |

** Kernel config

- =CONFIG_BLK_DEV_NVME=m= (must match Arch for initramfs)
- =CONFIG_NVME_HWMON=y= (thermal telemetry -- use on bench)
- =CONFIG_NVME_MULTIPATH=y= (single XG8 on this machine; **likely unnecessary**;
  see pass 8 in =docs/candidates.org=; consider =m= without default enable)

Initramfs: =nvme= only. See boot-safety doc.

Live root: btrfs on LUKS with =discard=async=; cmdline includes LUKS discard and
=resume_offset= for partial hibernate wiring (=linux-rg-7pnt=, =linux-rg-2ih0=).
