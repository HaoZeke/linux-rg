* Machine (rgam5terra profile)

Target machine:

- Fractal Design Terra mini-ITX SFF desktop.
- CPU: AMD Ryzen 9 9950X, Zen 5 (Granite Ridge), 16 cores / 32 threads, AM5.
- dGPU: ASUS GeForce RTX 5070 Dual 12 GB GDDR7 (Blackwell, GB205 class).
- Motherboard: ASRock B650I Lightning WiFi (AMD B650).
- Storage: WD BLACK SN850X 2 TB NVMe (PCIe Gen4).
- RAM: 48 GiB DDR5-8000 (2x24 GiB).
- Cooling: Noctua NH-L9a-AM4 low-profile physically mounted on AM5; no separate mounting kit assumed; ARCTIC P12 Pro 120 mm.
- PSU: Lian Li SP850 V2 850 W (ATX 3.1).

Planned inventory and acceptance thresholds: =docs/rgam5terra-targets.org=.

The package starts from Arch's =linux= package (kernel.org + Arch
=7.0.12.arch1= release). Per-profile overlay =rgam5terra.config= carries the
AM5/NVIDIA/Realtek profile and is selected with =LINUX_RG_PROFILE=rgam5terra=.

* Platform expectations (Zen 5 + discrete NVIDIA)

** CPU frequency and power

- Default cpufreq driver on modern AM5: =amd-pstate= (CPPC), not
  =acpi-cpufreq=.
- Kernel documentation:
  https://docs.kernel.org/admin-guide/pm/amd-pstate.html
- EPP (Energy Performance Preference) and preferred-core ranking are exposed via
  sysfs; desktop tuning is expected in userspace (EPP, PPT limits, fan curves).
- =CONFIG_X86_AMD_PSTATE=y= is the baseline expectation for this overlay.

** Scheduler topology

- 9950X is **homogeneous within each CCD** but has **two CCDs** with separate
  L3 caches. This is not Intel hybrid (P/E); the rgx1gen11 sched weld
  (0010-0013) does not apply by motivation.
- Cache-aware placement (0020) and sched_ext experts are **more relevant** here
  than on the laptop profile. Mainline and LPC materials on sched_ext are grade
  B; specific policy choice remains grade D until benchmarked.
- Server-centric load-balance criticism (Lozi et al., EuroSys 2016) is weakly
  relevant: 32 threads is large for a desktop but not a multi-socket server.

** Graphics

- No AMD iGPU on 9950X. Display and compute are NVIDIA-owned.
- Arch Linux documents Blackwell-and-newer GPUs on the open kernel module stack
  (=nvidia-open=), not the legacy proprietary-only path:
  https://wiki.archlinux.org/title/NVIDIA
- Early boot needs a policy for NVIDIA DRM in initramfs (see boot-safety doc).

** Evidence desert (honest framing)

Same grading as rgx1gen11: literature supports PSI/MGLRU/BBR/BFQ-class choices
and questions default mq schedulers on ULL NVMe (HOTSTORAGE 2023). Downstream
fair-class patches (BORE, 0010-0013) remain grade D until measured on this CPU.

** Default network stack hypothesis

Same as rgx1gen11 until bench says otherwise:

- =CONFIG_TCP_CONG_BBR=y=, =CONFIG_DEFAULT_BBR=y=
- =CONFIG_NET_SCH_FQ=y=, =CONFIG_DEFAULT_FQ=y=

Desktop fixed Ethernet reduces roam urgency but VPN/tunnel RTT still benefits
from BBR literature (grade A/B). Record Cubic fallback if LAN bufferbloat appears.

** Memory management hypothesis

Carry the same PSI/MGLRU/zswap/zram/DAMON pins as rgx1gen11 unless 48 GiB probing
shows headroom makes zram sizing redundant:

- MGLRU, PSI, memcg: literature-backed (grade A/B).
- 0018 + 0019 weld: downstream (grade D) but addresses same pressure problem
  framing as laptop.
- zram size target: start ~16-24 GiB (see targets doc); not copy-paste laptop 15.5 GiB.

** Block layer

- BFQ default + 0007 contention patch: same HOTSTORAGE-motivated skepticism.
- ADIOS module only; compare =none= on SN850X before any default change.

** Subsystem detail

*** T1-T3: 9950X + NH-L9a in Fractal Terra (critical)

| Fact | Source | Grade |
| 9950X default TDP 170 W | AMD product spec | B |
| NH-L9a-AM4 physically mounted on AM5 remains a 37 mm low-profile cooler class | Noctua product guidance + build note | B |
| Terra volume + RTX 5070 heat dump | build geometry | C (engineering) |

**Implication:** Linux scheduler or compiler wins are irrelevant if the CPU runs
uncapped and thermally throttles in a 95 W-rated cooler inside a closed SFF case.
**First tweak** is power (PPT/EDC/TDC in BIOS or =ryzenadj=), not another fair.c
patch.

Probe commands (post-assembly):

#+begin_src sh
# cpufreq driver and EPP
cat /sys/devices/system/cpu/cpufreq/policy0/scaling_driver
cat /sys/devices/system/cpu/cpu0/cpufreq/energy_performance_preference
# preferred cores (if present)
cat /sys/devices/system/cpu/amd_pstate_prefcore/status 2>/dev/null || true
# thermal throttle visibility
journalctl -k -b --no-pager -g 'thermal|Hardware Error|THRM'
#+end_src

Acceptance: sustained all-core load holds >= 4.0 GHz **or** documented intentional
PPT cap with stable temps under 95 C CPU / documented noise budget.

*** T12-T15: WD SN850X 2 TB

Primary sources: ArchWiki NVMe APST/NOPPM (B); Whitaker HOTSTORAGE 2023 (A).

rgx1gen11-nvme-check pattern (APST feature 0x0c, =default_ps_max_latency_us=,
=pcie_aspm.policy=) should be cloned as =rgam5terra-nvme-check= with SN850X
model/firmware constants after first =nvme list=.

WD-specific community reports of APST/resume quirks (grade C) justify **measure
first, require second**: do not hard-require APST enabled like rgx1gen11 until
SN850X passes five suspend cycles.

If APST is disabled, idle power rises (acceptable on AC desktop); latency on wake
may improve (HOTSTORAGE-style skepticism applies to scheduler, not APST alone).

** gaming-sched (sirlucjan 7.0) -- high-value for 32-thread desktop**

Intel maintainer series adding =cgroup_mode= sysfs for fair group scheduling:

| Mode | Behavior | Fit for 9950X + browser/build slices |
| smp | stock proportional shares per CPU | baseline |
| up | each CPU gets full tg->shares (ignore 1/n_cpus shrink) | **browser.slice / build.slice on 32 CPUs** |
| max | variant for max share accounting | bench after up |
| concur | concurrent group accounting | gaming + background compile |

Default in patch series: **up** mode. Touches =kernel/sched/fair.c= heavily;
**conflicts with BORE** same as newidle revert. Grade D until bench.

**Recommendation:** try **sched_ext** + ASA + cgroup slices first (already in
linux-rg). If PSI shows build.slice starvation on 32 CPUs, trial gaming-sched OR
newidle revert, **not both**, and only without adding CONFIG_CACHY fair.

* Decision gates (do not carry blindly)

1. **No hybrid sched weld** (0010-0011) without new evidence on Zen 5.
2. **No ADIOS default** without SN850X bench wins vs BFQ and =none=.
3. **No NVIDIA initramfs change** without encrypted-root and early-KMS probe on
   the actual display path.
4. **No zram shrink** below measured pressure needs on 48 GiB.
5. **Thermal caps trump scheduler wins**: a faster fair patch that raises sustained
   power in Terra is a regression for this profile.
6. **No laptop pcie_aspm=powersave** default without NVIDIA+SN850X resume matrix.
7. **No APST required** on SN850X until five clean suspend cycles with APST on.
8. **PPT cap before scheduler bench**: T1 must be stable before T5/T25 comparisons.
