#+title: linux-rg rgx1gen11 boot safety
#+date: [2026-06-18 Thu]

* Rule zero

linux-rg must boot to the same root filesystem as stock Arch for the active
profile. If linux-rg fails on a given machine, the stock =linux= kernel must
still start without manual recovery.

See =docs/profiles.org= for the full profile list.

This document records the boot contract for the rgx1gen11 profile. Other
profiles will have (or already have) their own boot-safety.org.

Vissue: =linux-rg-boot=. Parent: =linux-rg-mxx8=.

* What must not change (boot-critical)

| Item | Requirement | Why |
| Root disk driver | =CONFIG_BLK_DEV_NVME=m= (match Arch) | Root is on KIOXIA XG8 =nvme0n1= |
| GPU driver | =CONFIG_DRM_I915=m=, =CONFIG_DRM_XE=m= | Pick one via cmdline; both built as modules |
| Wi-Fi / BT | =CONFIG_IWLWIFI=m=, =CONFIG_BT_HCIBTUSB=m= | Load after init; need linux-firmware |
| Scheduler patches | Compile-tested only | BORE/0010-0013 must not block early boot |
| Firmware in kernel | =CONFIG_EXTRA_FIRMWARE=""= (Arch default) | Firmware stays in linux-firmware package |

** Never do for rgx1gen11

- Build =nvme=, =i915=, =xe=, or =iwlwifi= as built-in (=y=) unless Arch already does
  (it does not).
- Embed iwlwifi firmware into the kernel image.
- Enable =CONFIG_MQ_IOSCHED_DEFAULT_ADIOS= (ADIOS stays non-default module).
- Switch default DRM from i915 to xe without a tested cmdline on this panel.
- Drop =CONFIG_FW_LOADER= or =CONFIG_FW_LOADER_COMPRESS=.

* Before first linux-rg boot

1. Stock Arch kernel still installed: =pacman -Q linux linux-headers=.
2. linux-firmware installed: =pacman -Q linux-firmware= (AX211 Wi-Fi + BT firmware).
3. wireless-regdb installed if using regulatory domain helpers.
4. Successful =makepkg= for linux-rg (=linux-rg-py5f=).
5. mkinitcpio preset run for linux-rg with the probed rgx1gen11 MODULES and
   hooks contract (see below).
6. rEFInd entry points at linux-rg *after* stock entry (stock stays default until
   verified).
7. The package payload contains the installed boot/live checkers, hibernate
   image-size unit, DKMS overlay helper, module-symvers helper, pacman hook,
   and DKMS overlay patches.
8. =rgx1gen11-boot-check --installed= passes after package install.

* mkinitcpio: rgx1gen11 MODULES and hooks (mandatory)

Root is a btrfs subvolume inside =/dev/mapper/luks=, unlocked from the
initramfs. The internal panel/passphrase path needs early =i915= on this
machine. Wi-Fi, Bluetooth, SOF, and Thunderbolt are *not* required to unlock or
mount root.

#+begin_example
# /etc/mkinitcpio-linux-rg.conf generated by scripts/rgx1gen11-install-refind-root
MODULES=(nvme i915)
# Do NOT add iwlwifi btusb thunderbolt snd_sof unless root moves behind them.
#+end_example

Hooks: preserve the host's encrypted btrfs resume path:
=base udev plymouth modconf keyboard autodetect kms keymap consolefont block
encrypt btrfs resume filesystems fsck=.

The LUKS prompt uses the vconsole keymap from the initramfs. This profile
requires =/etc/vconsole.conf= to contain =KEYMAP=colemak= and the
=keyboard keymap consolefont= hooks to stay present.

The installer probes the live machine and validates the linux-rg package payload
before package installation or boot-file writes. It refuses to continue unless
the package includes =rgx1gen11-live-check=, =rgx1gen11-boot-check=, the
hibernate image-size service, and the DKMS overlay repair assets, including
=linux-rg-module-symvers=. It also
refuses to continue unless stock rEFInd options include
=cryptdevice=, =root=/dev/mapper/luks=, =rootflags=subvol=@=,
=resume=/dev/mapper/luks=, =resume_offset=, =i915=, and
=pcie_aspm.policy=powersave=, unless =/= is currently btrfs on
=/dev/mapper/luks=, and unless the initramfs keymap path is Colemak-safe.

* Hibernate image-size policy

linux-rg ships =rgx1gen11-hibernate-image-size.service=. The installer enables
it so boot sets =/sys/power/image_size= to 8 GiB. This keeps suspend-to-disk
write volume bounded while staying below the btrfs swapfile capacity. The live
gate is:

#+begin_example
rgx1gen11-hibernate-check --live
#+end_example

* Crash evidence policy

linux-rg keeps pstore support in the machine config:
=CONFIG_PSTORE=y=, =CONFIG_PSTORE_COMPRESS=y=, =CONFIG_EFI_VARS_PSTORE=y=,
and modular =ramoops= / =pstore-blk=. EFI pstore stays disabled by default
(=CONFIG_EFI_VARS_PSTORE_DEFAULT_DISABLE=y=) until =linux-rg-krgl= decides the
backend policy. Rsyslog is a userspace forwarder; it should consume
journald/kmsg/pstore output after storage semantics are fixed under
=linux-rg-8da1= and =linux-rg-krgl=.

* Rollback procedure

1. At rEFInd, select stock Arch =linux= entry.
2. If boot menu unavailable: firmware boot manager -> EFI partition -> stock
   =vmlinuz-linux= via existing bootloader path.
3. Do not remove =linux= package until =linux-rg-mxx8= and =linux-rg-slp= pass.
4. Documented in =linux-rg-61pb= acceptance.

* First-boot verification sequence

Run in order; stop and rollback if any step fails.

1. Boot linux-rg -> login on internal display.
2. =uname -r= shows linux-rg localversion.
3. =lsmod | rg 'nvme|i915|iwlmvm|btusb|snd_sof'= all load on demand.
4. =dmesg | rg -i 'iwlwifi.*loaded|firmware.*ma-b0|Bluetooth'= no firmware -ENOENT.
5. =ping -c1 127.0.0.1= only; then Wi-Fi via iwd/iwctl (linux-rg-wifi).
6. =sudo rg-linux-rg-postboot-check= passes.
7. One =systemctl suspend= + resume (linux-rg-slp).
8. Only then set rEFInd default to linux-rg.

* Package modprobe drop-ins

linux-rg ships =/usr/lib/modprobe.d/rgx1gen11-*.conf= for Wi-Fi/BT coexistence.
These apply at module load and do not change the initramfs. Override in
=/etc/modprobe.d/= if needed.
